# توثيق مشروع نظام التصويت الإلكتروني (E-Voting System)

هذه الوثيقة تحتوي على شرح شامل للمشروع، التقنيات المستخدمة في بنائه، نظرة عامة على كافة الصفحات، وخطوات إعداد بيئة التشغيل على جهاز جديد خطوة بخطوة.

---

## 1. التقنيات المستخدمة (Tech Stack)
تم بناء هذا المشروع باستخدام أحدث تقنيات الويب لضمان السرعة، الأمان، وتجربة الاستخدام الممتازة:

* **إطار العمل الأساسي:** **Next.js 15 (أو 14)** باستخدام بيئة (App Router) لتقديم أداء عالٍ ودعم الـ Server-Side Rendering (SSR).
* **لغة البرمجة:** **TypeScript** لضمان خلو الكود من الأخطاء أثناء التطوير وكتابة كود قوي وموجّه (Type-safe).
* **تصميم الوجهة الأساسية (Styling):** **Tailwind CSS** لبناء واجهات المستخدم بسرعة، مع استخدام مكتبة **Shadcn UI** للمكونات الجاهزة (مثل Button, Card, Dialog، إلخ).
* **قاعدة البيانات والمصادقة (Backend/BaaS):** **Supabase** لتوفير:
  * قاعدة بيانات يتيحها PostgreSQL.
  * نظام مصادقة المستخدمين (Authentication).
  * حماية وتشفير الصلاحيات بناءً على دور المستخدم (Row Level Security - RLS).
* **أدوات إضافية:** `lucide-react` للأيقونات، و`sonner` للإشعارات المنبثقة.

---

## 2. هيكلة المشروع (Project Structure)
المنظومة مصممة بناءً على بنية `src/app` الخاصة بـ Next.js:
* `src/app/`: يحتوي على مسارات التوجيه (Routes) والصفحات الرئيسية.
  * `(auth)`: صفحات تسجيل الدخول والمصادقة.
  * `(protected)`: الصفحات المحمية الخاصة بالمدير والطالب (لا يمكن الدخول إليها بدون تسجيل الدخول).
  * `(public)`: الصفحات العامة المتاحة للجميع.
  * `api/`: ملفات الواجهة البرمجية (API Routes) لمعالجة بعض العمليات الخلفية إن وجدت.
* `src/components/`: المكونات المجزأة القابلة لإعادة الاستخدام (مثل Sidebar, Navbar, ومكونات واجهة المستخدم UI).
* `src/lib/`: الملفات المساعدة ومكتبات الاتصال الخاصة بـ Supabase (مثل `client.ts` و `server.ts`).
* `src/services/`: تحتوي على وظائف التعامل مع قواعد البيانات بشكل منظم (مثل `admin.ts`, `candidate.ts`, `election.ts`, `profile.ts`).

## 3. شرح صفحات المشروع بالكامل

### أ. الصفحات العامة (Public Pages)
1. **الصفحة الرئيسية (`(public)/page.tsx`):** صفحة الهبوط التي تعرض نظرة عامة عن نظام التصويت.
2. **صفحة القواعد والإرشادات (`(public)/rules/page.tsx`):** تحتوي على شروط التصويت، قوانين الانتخابات، وإرشادات للمرشحين والناخبين.
3. **المرشحون العامون (`candidates/page.tsx`):** قائمة بأسماء المرشحين وبرامجهم الانتخابية تتاح للجمهور قبل أو أثناء الانتخابات.
4. **النتائج العامة (`results/page.tsx`):** صفحة تعرض إحصائيات ونتائج الانتخابات بعد انتهائها.

### ب. صفحات المصادقة والحسابات (Auth Pages)
1. **تسجيل الدخول (`login/page.tsx`):** تتيح للمستخدمين (طلاب وإداريين) إدخال البريد وكلمة المرور.
2. **إنشاء حساب (`signup/page.tsx`):** تتيح للطلاب الجدد تسجيل حساب في النظام (يتطلب تأكيد).
3. **تأكيد الرمز (`verify-otp/page.tsx`):** صفحة لتأكيد الحساب عبر رقم سري (OTP) يُرسل للبريد.
4. **نسيان كلمة المرور (`forgot-password/page.tsx`):** لاستعادة الحساب في حال فقدان كلمة السر.

### ج. لوحة تحكم الإدارة (Admin Dashboard) - `(protected)/admin/`
1. **الرئيسية (`dashboard/page.tsx`):** نظرة إحصائية عامة للانتخابات للمدير (أعداد الناخبين، نسب التصويت الحالية).
2. **إدارة المستخدمين (`users/page.tsx`):** الموافقة على حسابات الطلاب، حظر المخالفين، ومنح الصلاحيات، وإضافة الطلاب لقاعدة البيانات.
3. **إدارة المرشحين (`candidates/page.tsx`):** قبول أو رفض طلبات الترشح المقدمة من الطلاب للانتخابات.
4. **إدارة الطعون/الشكاوى (`appeals/page.tsx`):** مراجعة الطعون المقدمة من الطلاب (سواء على النتائج أو العملية الانتخابية) والرد عليها.
5. **النتائج والمراقبة (`dashboard/results/page.tsx`):** مراقبة تقدم التصويت بشكل مباشر والتحكم في ظهور النتيجة.
6. **التقارير (`reports/page.tsx`):** استخراج تقارير شاملة للانتخابات (PDF، أو ملفات Excel) ببيانات الناخبين والنتائج.

### د. منصة الطالب (Student Portal) - `(protected)/student/`
1. **الرئيسية (`dashboard/page.tsx`):** شاشة ترحيبية تعرض حالة الانتخابات الحالية وإشعارات مخصصة للطالب.
2. **الملف الشخصي (`profile/page.tsx` و `profile/edit/page.tsx`):** عرض البيانات الشخصية (الرقم الجامعي، الاسم، التخصص) مع إمكانية تحديث بعض المعلومات المسموح بها.
3. **التصويت (`vote/page.tsx`):** المنصة الرئيسية التي يتم من خلالها اختيار المرشحين بطريقة آمنة لا تسمح بالتصويت المتكرر.
4. **الترشح (`nominate/page.tsx`):** واجهة تتيح للطالب تقديم طلب لترشيح نفسه للانتخابات مع رفع برنامجه الانتخابي وملفاته.
5. **المرشحون (`candidates/page.tsx`):** تصفح المرشحين الحاليين ومقارنة برامجهم الانتخابية.
6. **تقديم الطعون (`appeals/page.tsx`):** تتيح للطالب تقديم شكوى أو طعن إلكتروني للإدارة ومتابعة حالته.

## 4. كيفية إعداد وتشغيل المشروع على جهاز جديد دائمًا (Setup Guide)

لتشغيل هذا المشروع لاحقًا على أي جهاز كمبيوتر جديد، اتبع الخطوات بدقة:

### متطلبات النظام الأساسية (Prerequisites):
1. تثبيت **Node.js** (يفضل إصدار LTS 20 فما فوق).
2. تثبيت **Git** لإدارة الكود (أو استخراج المشروع من الملف المضغوط).
3. محرر أكواد، مثل **Visual Studio Code**.

### خطوات الإعداد والتشغيل:

**الخطوة 1: تنصيب حزم الاعتمادية (Dependencies)**
افتح موجه الأوامر (Terminal) داخل مجلد المشروع (المسار المرادف لـ `voting-system`) ونفذ الأمر التالي:
```bash
npm install
```

**الخطوة 2: ربط قاعدة البيانات (Supabase Integration)**
بما أن المشروع يعتمد على Supabase، يجب إنشاء ملف `.env.local` في الجذر الرئيسي للمشروع (بجوار ملف `package.json`).
أضف المتغيرات التالية داخل الملف (ستحصل على هذه القيم من لوحة تحكم Supabase الخاصة بك > Settings > API):
```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project-id.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key
```

**الخطوة 3: تهيئة قاعدة البيانات (Database Setup)**
يحتوي المشروع على ملفات SQL جاهزة في المسار `src/lib/supabase/schema/` بالإضافة إلى ملف مساعد `clean_setup_and_seed.sql`.
1. اذهب إلى منصة Supabase، افتح مشروعك وانتقل إلى الـ **SQL Editor**.
2. انسخ محتويات الملف `clean_setup_and_seed.sql` وقم بتشغيلها هناك، لتهيئة الجداول (Users, Votes, Elections...) وتعيين صلاحيات الوصول وإضافة مستخدم Admin افتراضي للمرور إلى لوحة تحكم المدير.

**الخطوة 4: تشغيل خادم التطوير (Run the App)**
يمكنك الآن تشغيل واجهة المشروع على وضع التطوير عبر الأمر التالي في الـ Terminal:
```bash
npm run dev
```
*ملاحظة: يحتوي المشروع على ملف موجه أوامر مصغر بعنوان `run_app.bat` يمكنك الضغط عليه مرتين (Double click) من نافذة الويندوز للتشغيل التلقائي.*

**الخطوة 5: تصفح المشروع**
افتح المتصفح واذهب إلى الرابط:
`http://localhost:3000`
